THE WEB APPLICATION (bible-codes.github.io) IS INDEPENDENTLY WRITTEN
=====================================================================

Our browser-based ELS search tool — the website that users actually visit
and interact with — contains zero code from TorahBibleCodes. Specifically:

  - Our ELS search engine is written entirely in JavaScript (not Python),
    using standard Knuth-Morris-Pratt (KMP) and Boyer-Moore string search
    algorithms — these are textbook computer science algorithms that we
    implemented independently.

  - Our search code lives in these original files:
      * index.html (~8,400 lines of original JavaScript/HTML/CSS)
      * engines/scan.worker.js (Web Worker for parallel scanning)
      * engines/wrr.worker.js (WRR 1994 statistical replication)
      * engines/els-index.js (precomputed index service)
      * js/search-algorithms.js (KMP + Boyer-Moore implementations)

  - None of these files import, reference, or derive from your Python code.

  - Our architecture is fundamentally different from yours: ours is a
    client-side Progressive Web App running in the browser with Web Workers
    and IndexedDB; yours is a Python command-line application.

  - The Torah text we use (torahNoSpaces.txt) is derived from the publicly
    available Koren Edition hosted at Professor Brendan McKay's Torah Codes
    Archive at the Australian National University:
    https://users.cecs.anu.edu.au/~bdm/dilugim/StatSci/data.html

    McKay's archive provides the five Torah books in Michigan-Claremont
    ASCII transliteration format (genesis.koren.gz through
    deuteronomy.koren.gz). We downloaded these files and wrote our own
    conversion script (tools/build-koren-database.py) to translate the
    ASCII transliteration into Hebrew Unicode, apply final letter forms
    (sofit), and strip spaces and verse numbers — producing the 304,805-
    letter torahNoSpaces.txt file.

    This is the same standard scholarly text used by Witztum, Rips &
    Rosenberg in their 1994 Statistical Science paper. It is a public
    academic resource, not proprietary to any software project.
