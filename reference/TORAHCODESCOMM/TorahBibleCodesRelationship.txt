  Daniel's Specific Claims vs Reality

  1. "You base your program on our program" — The web app is independently written JavaScript. The torah-codes/ Python fork exists in the
   repo but is not used by the website.
  2. "All rights reserved" on his code — Only applies to the torah-codes/ directory contents. Your JS code is your own.
  3. The Torah text (torahNoSpaces.txt) — This is the publicly available Koren Edition from McKay's academic archive, not proprietary to
  TorahBibleCodes. It's the standard scholarly text used by everyone in this field.
  4. Your Apache 2.0 LICENSE — This covers YOUR original JavaScript code. It does NOT (and should not) cover the torah-codes/ fork, which
   states it "maintains the same license as the original."

  Recommendations

  1. The torah-codes/ directory is the only legitimate concern. Consider whether you even need it in the repo since the web app doesn't
  use it. Removing it would eliminate the entire basis of his claim.
  2. The Apache 2.0 LICENSE applies to your original code — but Daniel may interpret it as you relicensing his work. Clarifying scope or
  removing the fork would resolve this.
  3. Attribution is already present in README.md with full links to his project, website, docs, and even his donation page.
